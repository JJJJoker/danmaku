# 窗口干扰问题修复 - 最终方案

## 修复内容

已彻底修改控制面板的鼠标穿透策略，从"基于鼠标悬停自动切换"改为"仅在点击展开时临时启用交互"。

### 核心改进

**之前的问题**：
- 鼠标进入面板区域时立即调用 `setIgnoreMouseEvents(false)`
- 每次鼠标悬停都会触发模式切换
- Windows 系统将模式切换视为活动变化，通知其他应用
- 视频播放器检测到系统事件而暂停

**现在的方案**：
- 移除所有 `onMouseEnter` 和 `onMouseLeave` 监听
- 默认状态：窗口永远保持鼠标穿透
- 只有点击齿轮图标展开面板时，才调用一次 `setIgnoreMouseEvents(false)`
- 关闭面板时立即恢复穿透状态
- 大幅减少了模式切换频率（从每次悬停变为仅展开/关闭时）

## 如果仍有问题

### 问题 1：点击展开时视频仍然暂停

**说明**：这是预期的行为，因为 `setIgnoreMouseEvents(false)` 本身就是一个重量级操作。

**解决方案**：
- 接受这个限制：展开时会有一次干扰，但之后操作不会再有影响
- 或者考虑使用全局快捷键代替鼠标交互（终极方案）

### 问题 2：操作控件时视频持续暂停

**可能原因**：
- 旧版本缓存
- 修改未生效

**解决方案**：
1. 确认运行的是最新版本（检查打包时间）
2. 完全退出应用后重新启动
3. 检查控制台是否有错误

### 问题 3：拖拽时也影响视频

**说明**：拖拽时需要临时允许交互，这是必要的。

**解决方案**：
- 如果希望拖拽也完全不干扰，可以考虑移除拖拽功能
- 或者通过其他方式移动面板（如键盘快捷键）

## 技术说明

### Windows 焦点管理机制

- 当应用的输入模式发生变化时，Windows 会发送 WM_ACTIVATEAPP 等消息
- 其他应用收到这些消息后可能会调整自己的行为（如暂停视频）
- Electron 的 `setIgnoreMouseEvents()` 会触发这种模式变化
- **减少模式切换频率可以显著降低干扰**

### Electron 最佳实践

对于透明覆盖窗口：
- 尽量保持固定的输入模式
- 避免频繁的 `setIgnoreMouseEvents` 切换
- 如果必须切换，尽量缩短非穿透状态的持续时间
- 优先使用点击触发而非悬停触发

### 为什么这个方案更好

1. **最小化切换次数**：从"每次悬停"变为"仅展开/关闭时"
2. **可预测的行为**：用户知道什么时候会有影响（点击展开时）
3. **快速恢复**：关闭后立即恢复穿透状态
4. **用户体验平衡**：保留了完整的鼠标交互功能，同时大幅减少干扰
