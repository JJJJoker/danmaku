# 双窗口架构验证指南

## 架构说明

应用现在使用两个独立的 BrowserWindow：

1. **弹幕窗口（Danmaku Window）**：
   - 全屏透明覆盖
   - 永远保持鼠标穿透（`setIgnoreMouseEvents(true)`）
   - `focusable: false` - 永不获取焦点
   - `alwaysOnTop(true, 'screen-saver')` - 最高层级
   - 只显示弹幕内容

2. **控制面板窗口（Control Panel Window）**：
   - 400x600 小窗口
   - 正常接收鼠标和键盘事件
   - `focusable: true` - 可以获取焦点
   - `alwaysOnTop(true, 'floating')` - 较低层级（在弹幕窗口之下）
   - 包含所有设置控件

## 技术细节

### 窗口配置对比

| 特性 | 弹幕窗口 | 控制面板窗口 |
|------|---------|------------|
| 尺寸 | 全屏 | 400x600 |
| 透明 | ✅ | ✅ |
| 鼠标穿透 | ✅ 永远穿透 | ❌ 正常接收 |
| 可获取焦点 | ❌ focusable: false | ✅ focusable: true |
| alwaysOnTop | ✅ screen-saver | ✅ floating |
| skipTaskbar | ✅ true | ❌ false |
| 显示内容 | DanmakuLayer | ControlPanel |

### URL 参数

- 弹幕窗口：`?window=danmaku`
- 控制面板窗口：`?window=control`

App.tsx 根据这个参数决定渲染哪个组件。

### 焦点隔离原理

**之前的问题**：
- 单个窗口同时显示弹幕和控制面板
- 操作控制面板时需要调用 `setIgnoreMouseEvents(false)`
- 这个操作会触发 Windows 系统事件，导致背后视频暂停

**现在的解决方案**：
- 弹幕窗口永远穿透，永不获取焦点
- 控制面板是独立窗口，有自己的焦点管理
- 操作控制面板时，焦点只在控制面板窗口内
- 弹幕窗口背后的应用不受影响

## 常见问题

### Q1: 为什么看到两个窗口？

A: 这是新的双窗口架构设计。弹幕窗口是全屏透明的，您可能看不到它，但它确实在最顶层。控制面板是小窗口，可以正常交互。

### Q2: 控制面板挡住了弹幕怎么办？

A: 控制面板使用的是 `floating` 层级，低于弹幕窗口的 `screen-saver` 层级。理论上弹幕应该在控制面板之上。如果看到相反的情况，可能是窗口创建顺序问题，可以尝试重启应用。

### Q3: 如何完全退出应用？

A: 右键点击系统托盘图标，选择"退出"。这会关闭所有窗口并退出应用。

### Q4: 操作控制面板时视频还是暂停了？

A: 这不应该发生。请确认：
1. 您运行的是最新打包的版本
2. 视频是在弹幕窗口背后（不是在控制面板窗口背后）
3. 检查日志文件查看是否有错误

### Q5: 两个窗口的位置不协调怎么办？

A: 您可以自由拖动控制面板窗口到喜欢的位置。位置会在下次启动时保持（如果实现了位置保存）。
