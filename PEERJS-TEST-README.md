# PeerJS 连通性测试工具 - 使用说明

## 重要提示

这个测试工具需要**两个文件**一起使用:
1. `peerjs-test-standalone.html` - 测试页面
2. `peerjs-bundled.js` - PeerJS库文件

**必须确保这两个文件在同一个文件夹中!**

## 使用方法

### 步骤1: 准备文件

确保你有以下两个文件在同一目录:
- peerjs-test-standalone.html
- peerjs-bundled.js

### 步骤2: 主机方操作

1. **双击打开** `peerjs-test-standalone.html`
2. 点击"**创建房间**"按钮
3. 等待显示房间ID(例如: test-abc123)
4. **复制完整的房间ID**发给客户端

### 步骤3: 客户端操作

1. **把这两个文件都发给你的朋友**:
   - peerjs-test-standalone.html
   - peerjs-bundled.js
   
2. 朋友收到后,**确保两个文件在同一文件夹**

3. 朋友**双击打开** `peerjs-test-standalone.html`

4. 在输入框中**粘贴你给的房间ID**

5. 点击"**加入房间**"按钮

### 步骤4: 验证连接

如果连接成功:
- 双方都会看到绿色的"已连接"提示
- 可以点击"发送测试消息"进行双向通信测试
- 日志区域会显示收发消息的记录

如果连接失败:
- 查看日志区域的红色错误信息
- 按F12打开浏览器控制台,截图错误信息

## 常见问题

### Q: 显示 "Peer is not defined"
A: 说明 `peerjs-bundled.js` 文件缺失或不在同一目录。请确保两个文件在一起。

### Q: 一直显示"连接中"
A: 可能的原因:
1. 防火墙阻止了WebRTC连接 - 暂时关闭防火墙测试
2. NAT类型过于严格 - 已配置TURN中继,应该能解决
3. 房间ID输入错误 - 确认完全一致

### Q: 连接超时
A: 
1. 确认主机是否还在运行(没有关闭页面)
2. 确认房间ID完全正确
3. 检查网络连接是否正常

## 技术说明

- 使用了免费的公共TURN服务器(openrelay.metered.ca)
- 支持严格的NAT环境
- 无需安装任何软件,只需浏览器
- 所有数据通过加密连接传输

## 测试完成后

测试完成后可以删除这两个文件,不会影响你的主应用。
